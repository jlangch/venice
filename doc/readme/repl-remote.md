# Remote REPL 

The remote REPL is a Venice network based REPL that provides a REPL server and client for interactive inspection and control of remote Venice applications.

The communication between the local REPL and the remote REPL server is transparently 
AES-256 GCM (AES Galois Counter Mode) encrypted. The secret for establishing an AES-256
encryption is generated by the Diffie-Hellman key exchange algorithm.

Venice applications are *not* aware of the REPL server. No specific preparations are required.

 
 

## Usage 

* Start Venice application with a REPL server attached
* Connect to the server using the standard Venice REPL 
* Start inspection and control of the remote Venice application!

 

## Start Venice Application with REPL server

To attach a REPL server to a Venice application just add two launch parameters to set a port and a password:


| Option                 | Mandatory   | Description              | Example                |
| :--                    | :--:        | :--                      | :--                    |
| `-repl-port {port}`      | yes         | REPL communication port  | `-repl-port 333334`     |
| `-repl-pwd {password}`   | yes         | REPL password            | `-repl-pwd xcf6zu=UI`    |


    * `-repl-port {port}`
    * `-repl-pwd {password}`

 

Start the application with the REPL server port and password:

```text
java -jar ./venice-1.12.82.jar -repl-port 33334 -repl-pwd 123 -file ./remote-repl-demo.venice
```

 

The remote REPL demo application looks like:

```clojure
(do
  (ns demo)

  (def- stop? (atom false))


  (defn stop [] 
    (reset! stop? true)
    (println "Stopping server..."))

  (defn sum [x y] 
    (println "Function 'sum' called")
    (+ x y))


  (println "Started server")

  ;; just sleep until we get stopped
  (while (not @stop?)
    (sleep 1000))

  (println "Stopped server"))
```

 

## Connect from a REPL

Start a REPL and connect to the remote REPL server (pass *host*, *port*, and *password*)

```
venice> !remote localhost 33334 123

remote>
```

call a remote function

```
remote> (demo/sum 3 4)
Function 'sum' called
=> 7
```

switch back to the local REPL environment

```
remote> !local

venice>
```

 

## Remote REPL demo session

The demo uses this Venice server application:

```clojure
(do
  (ns demo)

  (def- stop? (atom false))


  (defn stop [] 
    (reset! stop? true)
    (println "Stopping server..."))

  (defn sum [x y] 
    (println "Function 'sum' called")
    (+ x y))


  (println "Started server")
  
  ;; just sleep until we get stopped
  (while (not @stop?)
    (sleep 1000))

  (println "Stopped server"))
```


It offers two public functions that can be remotely called:

| Function         | Description            | Example                |
| :--              | :--                    | :--                    |
| `demo/sum`        | sum up two numbers     | `(demo/sum 2 3)`        |
| `demo/stop`       | stop the application   | `(demo/stop)`           |

Standard out/err text produced by the remote functions are routed back to the REPL
client and displayed.
